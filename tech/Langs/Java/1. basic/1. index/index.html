    • Golang 之前没有泛型真的很麻烦，因为这就意味着外界一定得知道入参的基本结构，然后 reflect 强行转换，一转换不上就得报错了。

    E.g. 将 ES 文档对象按照规则转为DTO

    public static <T> T doc2DTO(BaseDocument doc, Class<T> clazz) {  
    //将文档对象转为Map  
    Map<String, Object> valueMap = BeanUtil.beanToMap(doc, false, false);  

    //创建DTO实例  
    T dto = ReflectUtil.newInstance(clazz.getName());  

    Field[] fields = ReflectUtil.getFields(clazz);  // DTO 需要有的字段

    for (Field f : fields) {  
        //填充属性值  
        fillFieldValue(dto, f, valueMap);  
    }  
    return dto;  
}

private static <T> void fillFieldValue(T t, Field f, Map<String, Object> valueMap) {  
    ESConfig fieldCfg = f.getAnnotation(ESConfig.class);  
    if (Objects.isNull(fieldCfg)) {  
        return;  
    }  
    //处理普通字段  
    if (StrUtil.isNotBlank(fieldCfg.column())) {  
        Object val = valueMap.get(fieldCfg.column());  
        if (Objects.nonNull(val)) {  
            ReflectUtil.setFieldValue(t, f, val);  
        }    } else if (!fieldCfg.fieldClass().isAssignableFrom(Void.class)) {  
        //处理自定义对象类型字段  
        ReflectUtil.setFieldValue(t, f, fillFieldValue(fieldCfg.fieldClass(), valueMap));  
}}

private static Object fillFieldValue(Class<?> clazz, Map<String, Object> valueMap) {  
    Object obj = ReflectUtil.newInstance(clazz.getName());  
    Field[] fields = ReflectUtil.getFields(clazz);  

    for (Field f : fields) {  
        //填充属性值  
        fillFieldValue(obj, f, valueMap);  
    }  
    return obj;  
}

    网络编程

    使用Socket进行网络编程时，本质上就是两个进程之间的网络通信。其中一个进程必须充当服务器端，它会主动监听某个指定的端口，另一个进程必须充当客户端，它必须主动连接服务器的IP地址和指定端口，如果连接成功，服务器端和客户端就成功地建立了一个TCP连接，双方后续就可以随时发送和接收数据。 - 对服务器端来说，它的Socket是指定的IP地址和指定的端口号； - 对客户端来说，它的Socket是它所在计算机的IP地址和一个由操作系统分配的随机端口号。
